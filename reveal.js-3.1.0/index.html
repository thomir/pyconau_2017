How we ship code is important.

Important, but often overlooked by engineers.

Day to day involves CODE! not CUSTOMERS.

Code isn't valuable until users are using it.

If you ship a software product, do you know what %ge of your users are on the latest version?

If it's significant, adding features may not be a good investment.

You will ship insecure software.

It's inevitable, not because you're bad engineers.

The sheer complexity of software, and depth of tech stack make this increasinglky likely.

I belieev companies that ship software have a MORAL obligation to keep their users safe.

Mirai botnet targetted IoT devices.

Which it turns out the Manufacturer cannot patch.

WannaCry did not target IoT, but many victims were still being targetted weeks after Microsoft released a patch.

Any update strategy that relies on users taking action to keep things up to date WILL fail.

How many of you have updated your ADSL router in the last 6 months?

Goals:

  • Secure by Default
  • Easy to adopt
  • Targets IoT, Desktop, Server, Cloud...
